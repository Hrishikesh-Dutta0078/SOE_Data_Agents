A: Narrative Flow

Insight text leads the response, followed by an inline sparkline/mini-chart. Table, SQL, entities, and classification are collapsed into small pills at the bottom — expandable on click. No tabs. The response reads like a brief executive report.

Pros

  • Most executive-friendly — answer first, details later
  • Minimal scrolling for the key info
  • Clean, report-like feel

Cons

  • Charts/tables require a click to expand
  • Less at-a-glance data for analysts

B: Split Pane

Chat stays narrow on the left, a detail panel opens on the right when results arrive. The insight narrative lives in the chat; chart + table + SQL live in the side panel. Clicking a different message switches the panel content.

Pros

  • See insights AND chart/table simultaneously
  • Chat history stays readable, not bloated by results
  • Easy to compare across questions

Cons

  • More complex layout — narrower chat on smaller screens
  • Bigger departure from the current design

C: Accordion Cards

Each response is a compact card with a headline + insight summary. Chart, table, SQL, and entities are accordion sections that expand inline. Only one section open at a time. Badges (confidence, row count) sit in the card header, not as separate rows.

Pros

  • Familiar single-column chat feel
  • Compact by default, details a click away
  • Easiest to implement — closest to current structure

Cons

  • Still vertical stacking — just more compact
  • Can't see chart + table simultaneously

My Recommendation: Approach A (Narrative Flow)

Best match for your executive-first goal. Insights lead, everything else is progressive disclosure. Keeps the single-column chat simplicity while dramatically reducing visual clutter. We can borrow the inline mini-chart from B's side panel idea without the layout complexity.
